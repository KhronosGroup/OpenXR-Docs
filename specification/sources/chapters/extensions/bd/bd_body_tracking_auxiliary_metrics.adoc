==== Overview

This extension enables applications to start the system's body tracking
calibration process, and to obtain additional status values and information
about body tracking.

Applications should: also enable the apiext:XR_BD_body_tracking extension to
get more data associated with body tracking.

==== Start body tracking calibration

[open,refpage='xrStartBodyTrackingCalibrationAppBD',type='protos',desc='Starts body tracking calibration',xrefs='']
--
The flink:xrStartBodyTrackingCalibrationAppBD function is defined as:

include::{generated}/api/protos/xrStartBodyTrackingCalibrationAppBD.adoc[]

.Parameter Descriptions
****
* pname:session is an slink:XrSession in which the body tracker will be
  active.
****

The application can: invoke the body tracking calibration app at any time
during any session's lifecycle.

The resulting body tracking calibration is managed in an implementation
defined way by the system.

Typically there is one current calibration applied globally to the entire
system.

Currently elink:XrBodyTrackingPostureBD is applicable to
ename:XR_BODY_JOINT_LEFT_ANKLE_BD and ename:XR_BODY_JOINT_RIGHT_ANKLE_BD (in
elink:XrBodyJointBD).

The runtime must: populate the data corresponding to the indices of
ename:XR_BODY_JOINT_LEFT_ANKLE_BD and ename:XR_BODY_JOINT_RIGHT_ANKLE_BD
with valid data and set all others to 0.
