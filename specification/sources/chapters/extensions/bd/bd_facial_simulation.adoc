==== Overview

This extension allows applications to get weights of blend shapes which
express the users' face movements.

This extension defines several modes in elink:XrFacialSimulationModeBD.
These modes correspond to different output types of blend shapes, such as
the elink:XrFaceExpressionBD or elink:XrLipExpressionBD, or even a
combination of them.

Because not all modes involve direct use of private data like images, and
some modes are closer to an audio-driven estimation of facial expression,
this extension is referred to as Facial Expression Simulation instead of
Facial Expression Tracking.

[IMPORTANT]
.Permissions
====
Android applications must: have the com.picovr.permission.FACE_TRACKING
permission and the android.permission.RECORD_AUDIO listed in their manifest
and granted to use this extension, otherwise flink:xrCreateFaceTrackerBD
function will return a ename:XR_ERROR_PERMISSION_INSUFFICIENT error.
(protection level: dangerous)
====

==== Inspect system capability

[open,refpage='XrSystemFacialSimulationPropertiesBD', desc='System property for facial simulation',type='structs', xrefs='']
--

The slink:XrSystemFacialSimulationPropertiesBD structure is defined as:

include::{generated}/api/structs/XrSystemFacialSimulationPropertiesBD.adoc[]

.Member Descriptions
****
* pname:type is the elink:XrStructureType of this structure.
* pname:next is code:NULL or a pointer to the next structure in a structure
  chain.
  For more details about the structure chain, see the structure being
  extended (slink:XrSystemProperties).
* pname:supportsFaceTracking is an basetype:XrBool32, indicating if the

An application can: inspect whether the system is capable of providing face

flink:xrGetSystemProperties.

If a runtime returns ename:XR_FALSE for pname:supportsFaceTracking, the
runtime must: return ename:XR_ERROR_FEATURE_UNSUPPORTED from
flink:xrCreateFaceTrackerBD.

==== Enumerating tracking modes

The application can: enumerate the supported modes by calling the
flink:xrEnumerateFacialSimulationModesBD function.
The list of available modes depends on the internal facial simulation
algorithm.


[open,refpage='xrEnumerateFacialSimulationModesBD', desc='Enumerate the facial simulation modes that runtime supports',type='protos']
--

The flink:xrEnumerateFacialSimulationModesBD function is defined as:

include::{generated}/api/protos/xrEnumerateFacialSimulationModesBD.adoc[]

.Parameter Descriptions
****
* pname:session is an slink:XrSession previously created with
  flinl:xrCreateSession.
* pname:modeCapacityInput is the capacity of the pname:modes array, or `0`
  to retrieve the required capacity.
* pname:modeCountOutput is a pointer to the count of pname:modes written, or
  a pointer to the required capacity in the case that
  pname:modeCapacityInput is insufficient.
* pname:modes is a pointer to an array of elink:XrFacialSimulationModeBD
  values, but can: be code:NULL if pname:modeCapacityInput is `0`.
* See the <<fundamentals-buffer-size-parameters>> section for a detailed
  description of retrieving the required pname:modes size.
****

include::{generated}/validity/protos/xrEnumerateFacialSimulationModesBD.adoc[]
--

This extension provides four modes to express facial blend shapes.
Some modes just need voice input for audio-driven facial expression
simulation.
Other modes require image input, making use of face tracking.

[open,refpage='XrFacialSimulationModeBD',desc='Facial simulation modes',type='enums']
--

The elink:XrFacialSimulationModeBD enumeration describes the modes of facial
simulation and tracking.

include::{generated}/api/enums/XrFacialSimulationModeBD.adoc[]

include::{generated}/api/enums/XrFacialSimulationModeBD.comments-box.adoc[]

When using ename:XR_FACIAL_SIMULATION_MODE_DEFAULT_BD and
ename:XR_FACIAL_SIMULATION_MODE_COMBINED_AUDIO_BD, only
slink:XrFacialSimulationDataBD is populated in the output.

When using ename:XR_FACIAL_SIMULATION_MODE_COMBINED_AUDIO_WITH_LIP_BD the
application must: chain an slink:XrLipExpressionDataBD to
slink:XrFacialSimulationDataBD::pname:next when calling
flink:xrGetFacialSimulationDataBD.

When using ename:XR_FACIAL_SIMULATION_MODE_ONLY_AUDIO_WITH_LIP_BD the
runtime may: generate predicted animations in face expression in
slink:XrFacialSimulationDataBD and slink:XrLipExpressionDataBD structures
when calling flink:xrGetFacialSimulationDataBD, to make the avatar look more
natural.

.Member Descriptions

[open,refpage='XrFacialSimulationDataBD', desc='contain the facial expression', type='structs', xrefs='']
--
The slink:XrFacialSimulationDataBD structure is defined as:

include::{generated}/api/structs/XrFacialSimulationDataBD.adoc[]

.Member Descriptions
****
* pname:type is the elink:XrStructureType of this structure.
* pname:next is code:NULL or a pointer to the next structure in a structure
  chain.
* pname:faceExpressionWeightCount is a code:uint32_t describing the count of
  elements in pname:faceExpressionWeights array.
* pname:faceExpressionWeights is a array of code:float that will be filled
  with weights of facial expression blend shapes.
* pname:isUpperFaceDataValid is an basetype:XrBool32 representing the
  validity of upper face, generally the eyes and its surrounding area
* pname:isLowerFaceDataValid is an basetype:XrBool32 representing the
  validity of lower face, generally the mouth and its surrounding area
* pname:time is the basetype:XrTime time at which the returned expression
  weights were tracked
****

The slink:XrFacialSimulationDataBD structure contains the facial expression
weight.
See elink:XrFacialSimulationModeBD for details and for how this interacts
with lip expression tracking.

The runtime must: fill the pname:faceExpressionWeights array ordered so that
it is indexed using the corresponding facial expression enum
elink:XrFaceExpressionBD.

An application must: preallocate the output pname:faceExpressionWeights
array that can contain at least pname:faceExpressionWeightCount of
code:float.
The value of pname:faceExpressionWeightCount must: be
dlink:XR_FACE_EXPRESSION_COUNT_BD.

* pname:type is the elink:XrStructureType of this structure.

that it is indexed using the corresponding lip expression enum
elink:XrLipExpressionBD.

array that can contain at least pname:lipsyncExpressionWeightCount of
code:float.
pname:lipsyncExpressionWeightCount must: be
dlink:XR_LIP_EXPRESSION_COUNT_BD.

==== Example code for getting facial expression and lip expression

The code below shows how to use this extension.

[source,cpp]
----
XrInstance instance; // previously initialized
XrSystemId systemId; // previously initialized
XrSession session;   // previously initialized

XrSystemFacialSimulationPropertiesBD facialSimulationProperties{
    XR_TYPE_SYSTEM_FACIAL_SIMULATION_PROPERTIES_BD};
XrSystemProperties sysProperties{XR_TYPE_SYSTEM_PROPERTIES,
                                 &facialSimulationProperties};
CHK_XR(xrGetSystemProperties(instance, systemId, &sysProperties));

if (facialSimulationProperties.supportsFaceTracking == false) {
  return;
}

uint32_t modeCount;
CHK_XR(xrEnumerateFacialSimulationModesBD(session, 0, &modeCount, nullptr));
std::vector<XrFacialSimulationModeBD> facialSimulationModes(modeCount);
CHK_XR(xrEnumerateFacialSimulationModesBD(session, modeCount, &modeCount,
                                          facialSimulationModes.data()));

XrFaceTrackerBD faceTracker{XR_NULL_HANDLE};
XrFaceTrackerCreateInfoBD info{XR_TYPE_FACE_TRACKER_CREATE_INFO_BD};
info.mode = XR_FACIAL_SIMULATION_MODE_DEFAULT_BD;
CHK_XR(xrCreateFaceTrackerBD(session, &info, &faceTracker));

XrFrameState frameState; // previously returned from xrWaitFrame
const XrTime time = frameState.predictedDisplayTime;

XrFacialSimulationDataGetInfoBD infoBd;
infoBd.type = XR_TYPE_FACIAL_SIMULATION_DATA_GET_INFO_BD;
infoBd.time = time;

XrFacialSimulationDataGetInfoBD getInfo{
    XR_TYPE_FACIAL_SIMULATION_DATA_GET_INFO_BD};
getInfo.time = time;
std::vector<float> faceExpressionWeights(XR_FACE_EXPRESSION_COUNT_BD);
XrFacialSimulationDataBD facedata{XR_TYPE_FACIAL_SIMULATION_DATA_BD};
facedata.faceExpressionWeights = faceExpressionWeights.data();

CHK_XR(xrGetFacialSimulationDataBD(faceTracker, &getInfo, &facedata));
assert(facedata.isUpperFaceDataValid == true);
assert(facedata.isLowerFaceDataValid == true);

for (auto blendWeight : faceExpressionWeights) {
  std::cout << blendWeight << '\n'; // print each data of face expression
}

// change mode from default to COMBINED_AUDIO_WITH_LIP
CHK_XR(xrSetFacialSimulationModeBD(
    faceTracker, XR_FACIAL_SIMULATION_MODE_COMBINED_AUDIO_WITH_LIP_BD));

std::vector<float> lipExpressionWeights(XR_LIP_EXPRESSION_COUNT_BD);
XrLipExpressionDataBD lipdata{XR_TYPE_LIP_EXPRESSION_DATA_BD};
lipdata.lipsyncExpressionWeights = lipExpressionWeights.data();

facedata.next = &lipdata;
CHK_XR(xrGetFacialSimulationDataBD(faceTracker, &getInfo, &facedata));

for (auto blendWeight : lipExpressionWeights) {
  std::cout << blendWeight << '\n'; // print each data of lip expression
}

XrFacialSimulationModeBD currentMode;
CHK_XR(xrGetFacialSimulationModeBD(faceTracker, &currentMode));
assert(currentMode == XR_FACIAL_SIMULATION_MODE_COMBINED_AUDIO_WITH_LIP_BD);

CHK_XR(xrDestroyFaceTrackerBD(faceTracker));
----
